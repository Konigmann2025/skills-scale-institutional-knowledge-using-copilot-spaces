# OctoAcme Project Management Docs

> **Purpose:** This documentation collection centralizes and standardizes OctoAcme's project management knowledge to support onboarding, improve process findability, and enable clearer team communication across all cross-functional projects.

---

## About This Collection

These documents describe how OctoAcme plans, executes, tracks, and closes projects. They are intended as living references for Project Managers, Product Managers, Developers, QA, and Stakeholders alike. Add the relevant docs to `.copilot/` in your project repository to make them available as context in Copilot Spaces.

---

## Core Principles & Values

OctoAcme's project management approach is grounded in five values:

- **Customer-first** — Prioritize customer value and usability in every decision.
- **Iterative delivery** — Ship small, testable increments frequently rather than large, risky releases.
- **Clear ownership** — Every project has a named Project Manager (PM) and Product Lead accountable for outcomes.
- **Data-informed decisions** — Measure impact and iterate based on evidence, not assumptions.
- **Psychological safety** — Encourage open feedback, candid retrospectives, and continuous learning.

## Project Lifecycle Overview

| Phase | Key Activities |
|---|---|
| **1. Initiation** | Problem statement, stakeholder alignment, high-level timeline, go/no-go decision |
| **2. Planning** | Scope breakdown, backlog creation, estimation, Definition of Done, release plan |
| **3. Execution** | Build, test, code review, iterate in sprints; track on project board |
| **4. Release** | Deploy to staging, smoke-test, deploy to production, announce to stakeholders |
| **5. Close & Retrospective** | Capture learnings, identify action items, measure improvement |

---

## Core Roles

| Role | Responsibilities |
|---|---|
| **Project Manager (PM)** | Coordinates delivery, schedules, risk, and communications; maintains project board and status reports |
| **Product Manager (PdM)** | Defines outcomes, prioritizes the roadmap and backlog, measures success metrics |
| **Developers** | Implement features, write tests, participate in design and code reviews, estimate work |
| **QA / Testing** | Validate quality, verify acceptance criteria, run integration and smoke tests |
| **Stakeholders** | Provide inputs, sign off on deliverables, receive regular status updates |

---

## Key Artifacts

- **Project Charter / One-pager** — Problem statement, goal (SMART), success metrics, stakeholders, timeline, risks
- **Roadmap & Release Plan** — Milestones and target dates aligned to sprint cadence
- **Sprint / Iteration Backlog** — Prioritized list of work items with acceptance criteria, estimates, and owners
- **Risk Register** — ID, description, impact, likelihood, owner, mitigation, and status
- **Definition of Done (DoD)** — Team-agreed quality bar for any shippable increment
- **Retrospective Notes & Action Items** — Learnings and owners tracked to completion

## Communication Cadence

| Meeting | Frequency | Participants |
|---|---|---|
| Standup | Daily (15 min) | Delivery team |
| Delivery Sync | Weekly | PM + PdM + Tech Lead |
| Sprint Demo / Review | End of each sprint | Team + Stakeholders |
| Stakeholder Update | Monthly | PM + Stakeholders |
| Retrospective | After each sprint, release, or incident | Full team |
| Ad-hoc escalation | As needed | PM → Product Lead → Sponsor |

## How to Use These Docs

1. **New team members:** Start with the [Project Management Overview](octoacme-project-management-overview.md) for a complete introduction, then read the role-specific guidance in [Roles & Personas](octoacme-roles-and-personas.md).
2. **Starting a project:** Follow the [Project Initiation Guide](octoacme-project-initiation.md) to validate the idea and produce a Project One-pager.
3. **Planning a sprint or release:** Use [Project Planning](octoacme-project-planning.md) and the [Release & Deployment Guide](octoacme-release-and-deployment.md).
4. **Day-to-day execution:** Refer to [Execution & Tracking](octoacme-execution-and-tracking.md) for workflow and metrics guidance.
5. **Managing risks:** Keep the Risk Register current using [Risk Management & Communication](octoacme-risks-and-communication.md).
6. **Closing a sprint or project:** Run a structured retrospective with [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md).
